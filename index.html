Back-end Code

Overview

MealMatcher was created with the idea that people should never have to eat alone. This application allows people to connect with food. It is useful for both travelers and locals looking to experience new restaurants. It is a full-stack application where the front-end is built with Angular4 and the back-end is run on nodeJS.

A user can search for a city they live in or one they are traveling to. The app will find a list of the top 20 restaurants in that location. Signing up allows them to be contacted when someone is interested in eating there with them or to contact someone if there are already interested parties.

Tech notes

This was my final project for Thinkful, built in a one man team.

All user related data and login systems are stored on a personal database, while the restaurant data is from Zomato’s API. Images are complements of both The Google Places API and Google’s Geolocation API. Map data is provided by Google Maps. Google’s GeoLocation API also assists in providing accurate restaurant searches by converting search results into latitude and longitude GPS coordinates.

Challenges

Tying five external API’s and 2 local databases proved to be quite the task.

This was my first project with angular so the learning curve presented some difficulties.

Upcoming Features

Being that there is a lot of potential for improvement, I will use this project as a springboard to learn as much about Angular as I can.

Potential enhancements: Calendar system would be quite nice to implement, a profile page so you can get to know the other person before you go to eat with them, an in app messing system to replace the current system, and a total redesign for the UI of the app.

This provides to be a good project to learn about Ionic and conversion into a mobile app.
